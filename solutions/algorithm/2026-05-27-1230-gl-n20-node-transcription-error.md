# Solution: Gauss-Legendre n=20 Node Transcription Error

**Date**: 2026-05-27T12:30:00Z
**Category**: algorithm
**Related Issue**: #402

## Problem

The test `should integrate x^2 over [0,1] exactly for n=20` failed with an integration error of ~9.77e-7, far larger than the expected sub-epsilon residual (`Number.EPSILON * 10 ≈ 2.2e-15`). The n=5 and n=10 tests passed cleanly. The failure indicated something was wrong with the n=20 node or weight table.

## Root Cause

The n=20 nodes and weights were hand-transcribed from a printed reference table (Abramowitz & Stegun Table 25.4 as republished on an external site). The node at pair-index 5 was stored as `±0.7463062256567499` when the correct value is `±0.7463319064601508` — a difference of ~2.6e-5 in the fourth decimal place. The values look superficially similar and escaped manual review.

Eight of the ten n=20 weights also carried 1-2 ULP errors for the same reason: the reference's printed precision was insufficient to pin down all 16 significant figures of an IEEE-754 double.

These errors do **not** fail a weight-sum-to-2.0 sanity check (each per-entry ULP error rounds away in the sum) and do **not** fail a smooth-function accuracy test like `exp(x)` (the polynomial approximation error from the quadrature formula dominates the table inaccuracy). Only the **polynomial exactness** property exposes them: GL must give zero integration error for polynomials of degree ≤ 2n-1, and any table inaccuracy at double-precision scale shows up as a non-zero residual.

## Fix

All n=20 nodes and weights were replaced with values generated by Newton-Raphson iteration on the Legendre polynomial using the 3-term recurrence:

```js
function legendreP(n, x) {
  let p0 = 1, p1 = x
  for (let k = 2; k <= n; k++) {
    const p2 = ((2*k-1)*x*p1 - (k-1)*p0) / k
    p0 = p1; p1 = p2
  }
  return p1
}
```

Newton-Raphson converges to the correct root in ~5 iterations from the initial guess `cos(π*(4i-1)/(4n+2))`, reaching full double precision without relying on any external table. The weights are computed as `2 / ((1 - x²) * P'_n(x)²)`.

The corrected node: `±0.7463319064601508` (was `±0.7463062256567499`).
All n=20 weights corrected by 1-2 ULPs each.

Verification: after correction, `Σ w_i * x_i² = 2/3` to 3.3e-16 (machine epsilon range).

## Prevention Strategy

1. **Never copy GL tables for n ≥ 10 from printed sources.** Print references and web reproductions (including Abramowitz & Stegun) may not carry enough significant figures to fill all 16 digits of a double. Use the Legendre Newton-Raphson snippet above to generate constants programmatically.

2. **Always include a polynomial exactness test for every supported order.** The test `|GL(x², 0, 1, n) - 1/3| < EPSILON * 10` must be present for each n in {5, 10, 20, ...}. This is the only test that reliably catches both gross node errors and single-ULP weight errors simultaneously, because it exploits the theoretical zero-error guarantee of GL quadrature.

3. **Do not rely on smooth-function accuracy tests (exp, sin) as the primary correctness gate for table accuracy.** These tests can pass even when the table is wrong by ~2.6e-5 per node, because the quadrature approximation error from the non-polynomial integrand masks the table error.
